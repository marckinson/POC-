Upload File

In order to realize a mocked file management API, we have provided this extension in our demo / POC. This extension stores the file in back-end and provides directly the checksum obtained via the chaincode. Here, the supplied checkSum represents the sha512 of the file and is converted to Uint8Array in order to avoid encoding problems but it can be another hash function. The chaincode has a reference status on the hash function.

Ideally, we will use a file management API of the bank. This extension is used in the make payment instructions in order to save the file to better find it.

Note: we do not save the file in the chaincode only its hash/checksum (Uint8Array). In the demo context, the lifetime of the files is therefore limited to the back-end process node
