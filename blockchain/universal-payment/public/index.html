Proposal 7: Payment Instruction Message Directly Originated from Chain Code

Problem Description :


Currently, in Hyperledger, any value transaction that is the outcome of a Smart Contracts (Chain Code) is based in its own value token inside the Hyperledger blockchain implementation. There is currently no integration available between Hyperledger and an existing Payment Infrastructure.

This problem can be solved in two manners:
  1. Through an integration code that is called from inside the Chain Code to create a valid payment instruction. Then, connect this message to an existing banking provider following the rules of a specific Payment Network (ACH, SEPA, and SWIFT) for downstream processing
  2. Alternatively, through the integration of the Hyperledger blockchain with Interledger protocol (http://interledger.org). Thiswill allow Interledger to be the connection and distribution of the value transfer. It also allows for the Interledger protocol to do the discovery, routing, and value settlement between Hyperledger and the endÂ­destination ledger (this could be any participating payment network connected to Interledger).

Benefits :


This concept will solve one of the most critical components for private ledgers, namely the integration between the blockchain (Hyperledger) and traditional value (payment) systems for settlement. In this scenario, blockchain transactions could become for Financial Institutions a new channel for automated payment services.
